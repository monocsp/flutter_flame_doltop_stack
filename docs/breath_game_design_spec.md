# 민들레 호흡 게임 — 기획 분석서

> 이 문서는 기존 "민들레 호흡 여정" 게임의 전체 설계를 분석한 것입니다.
> 별자리 만들기 등 새로운 호흡 기반 게임을 기획할 때 참고용으로 작성되었습니다.

---

## 1. 전체 유저 플로우

```
인트로 (5페이지 스와이프) → 게임 (3라운드) → 결과 화면
```

### 인트로 화면 (세로 스와이프, 5페이지)

| 페이지 | 내용 | 역할 |
|--------|------|------|
| 1. 히어로 | "DANDELION BREATH" / "호흡으로 민들레 씨앗을 세상 끝까지 날려보세요" | 세계관 제시 |
| 2. 스토리 | "잠깐 멈춰 호흡해봐요" / "바람을 품은 민들레처럼, 당신의 숨결도 어딘가에 닿을 수 있어요" | 감성 동기부여 |
| 3. 방법 안내 | 4단계 설명 (마이크 가까이 → 들숨 3초 → 천천히 내쉼 → 3번 반복) | 인터랙션 학습 |
| 4. 마이크 권한 | "마이크는 게임 중에만 사용되며, 어디에도 저장되지 않아요" | 권한 요청 + 신뢰 |
| 5. 시작 | "준비됐나요? 편안한 자세로 천천히 시작해요" | 진입 |

**핵심 설계**: 스토리 → 동기 → 방법 → 신뢰 → 시작 순서로 감정을 쌓아감.

## 2. 게임 플레이 구조

### 4단계 순환 × 3라운드

```
[준비 3초] → [들숨 3초] → [날숨 0~5초] → [라운드 브레이크 2.2초] → 반복
```

#### Phase 1: 준비 (PREPARE) — 3초
- 카운트다운: 3 → 2 → 1
- 텍스트: "입을 마이크에 가까이 대세요"
- 마이크 아이콘 + 초록 원형 테두리
- **목적**: 유저가 자세를 잡고 마이크 위치를 확인하게 함

#### Phase 2: 들숨 (INHALE) — 3초
- 카운트다운: 3 → 2 → 1
- 텍스트: "숨을 크게 들이쉬세요"
- 배경에 민들레 SVG 표시
- **핵심**: 이 3초 동안 마이크로 **주변 소음을 측정**하여 기준점 설정 (캘리브레이션)
- "주변 소음을 측정하고 있어요…" 힌트 표시

#### Phase 3: 날숨 (EXHALE) — 최대 5초
- 텍스트: "내쉬세요"
- 진행 바로 경과 시간 표시
- **유저가 내쉬는 동안 실시간 시각 피드백** (아래 상세)
- 내쉬기를 멈추거나 5초 경과 시 자동 종료
- 2초 동안 미감지 시 힌트: "혹시 마이크에서 떨어져있거나, 막혀있지 않나요?"

#### Phase 4: 라운드 브레이크 — 2.2초
- "잘했어요!" (마지막 라운드면 "완료!")
- "씨앗 N개가 날아갔어요"
- 2.2초 후 다음 라운드 또는 결과 화면으로 이동

## 3. 호흡 감지 메커니즘 (핵심 기술)

### 입력 소스
- `noise_meter` 패키지 사용 (마이크 → 데시벨 측정)
- 초당 약 8~10회 dB 리딩

### 캘리브레이션 (매 라운드 들숨 3초 동안)
```
1. 들숨 3초간 dB 리딩 수집
2. 읽기 3개 이상: 중앙값(median)을 주변 소음 기준으로 설정
   읽기 1~2개: 평균값 사용
   읽기 0개: 폴백 52dB 사용
3. 감지 임계값 = 주변 소음 + 10dB
```

**왜 중요한가**: 방마다 소음이 다름 (에어컨, 선풍기, 카페). 하드코딩 대신 매 라운드 적응하여 어떤 환경에서든 작동.

### 호흡 감지 알고리즘

```
breathLevel = ((현재dB - 임계값) / 18.0).clamp(0.0, 1.0)

매 120ms마다:
  normalized = (현재dB - 임계값) / 18.0

  if normalized <= 0:
      연속감지 = 0, 호흡활성 = false
  else:
      연속감지++
      if 연속감지 >= 2:  (약 240ms 지속)
          호흡활성 = true → 씨앗 생성!
```

### 핵심 상수
| 상수 | 값 | 의미 |
|------|---|------|
| `ambientOffset` | 10.0 dB | 주변소음 대비 감지 오프셋 |
| `breathRange` | 18.0 dB | 강도 정규화 범위 (0→1) |
| `fallbackThreshold` | 52.0 dB | 캘리브레이션 실패 시 기본값 |
| `sustainedTicksRequired` | 2 (240ms) | 오탐 방지 최소 지속 시간 |

### 오탐 방지 설계
- **240ms 지속 필터**: 순간적 환경 소음 스파이크 무시
- **라운드별 재캘리브레이션**: 환경 변화에 대응
- **10dB 오프셋**: 주변 소음과 호흡 구분에 충분한 마진

## 4. 시각 피드백 시스템 (인터랙션 핵심)

### breathLevel (0.0~1.0)이 제어하는 요소들

| 요소 | 최소 (0.0) | 최대 (1.0) | 공식 |
|------|-----------|-----------|------|
| 씨앗 속도 | 60 px/s | 225 px/s | `60 + intensity × 130 + random × 35` |
| 씨앗 크기 | 0.3배 | 1.0배 | `0.3 + intensity × 0.5 + random × 0.2` |
| 씨앗 수명 | 2.2초 | 4.7초 | `2.2 + intensity × 1.5 + random × 1.0` |
| 민들레 흔들림 | ±3.4° | ±9.2° | `sin(t × 3.5) × (0.06 + level × 0.10)` |
| 배경 글로우 | 0% | 8% 알파 | `level × 0.08` |
| 감지 아이콘 크기 | 52px | 76px | `52 + level × 24` |
| 아이콘 글로우 | 15% 알파 | 40% 알파 | `0.15 + level × 0.25` |

### 4계층 렌더링 구조

```
[배경 글로우] → [파티클 레이어] → [씨앗 트레일] → [씨앗 + 민들레]
```

#### 1) 배경 글로우
- 호흡 감지 시 RadialGradient로 은은한 초록빛 (#7ADAA5)
- 중심: 민들레 위치보다 약간 위
- 반지름: 화면 폭 × 0.9

#### 2) 글로우 파티클
- 씨앗이 날아가면서 120ms마다 3개씩 생성
- 작은 원형, 위로 떠오르며 사라짐
- 색상: #E8E0D4 / #F7F3E9 (크림/아이보리 계열)
- 오브젝트 풀: 최대 120개 (GC 방지)

#### 3) 씨앗 트레일
- 각 씨앗의 최근 8개 위치를 원형 큐로 저장
- 뒤쪽일수록 투명하고 얇은 스트로크로 렌더링
- 씨앗이 날아가는 궤적을 시각화

#### 4) 날아가는 씨앗
- SVG 이미지를 ui.Image로 래스터화하여 Canvas에 직접 렌더 (성능)
- 물리 시뮬레이션: 바람(사인파 진동) + 중력 + 공기저항
- 회전: 바람력에 비례하여 자연스러운 회전

### 민들레 흔들림 (Sway)
```
oscillation = sin(경과시간 × 3.5)                    // 0.56Hz 주기
amplitude = 0.06 + breathLevel × 0.10              // 3.4° ~ 9.2°
swayTarget = oscillation × amplitude
실제sway += (swayTarget - 실제sway) × 4.0 × dt     // 부드러운 보간
```
- 바람에 흔들리는 느낌을 주면서, 호흡 세기에 따라 진폭 변화
- bottomCenter 기준으로 회전 (뿌리는 고정, 줄기가 흔들림)

### 씨앗 물리 시뮬레이션
```
매 프레임:
  바람 = sin(age × 2.5 + 랜덤위상) × 진폭     // 좌우 흔들림
  vx += 바람 × dt
  vy += 12.0 × dt  (수명 30% 이후부터, 약한 중력)
  vx *= 0.985  (공기저항)
  vy *= 0.985
  회전 += (바람 × 0.08 + 0.5) × dt
```

## 5. 결과 화면

### 연출 타임라인

```
0ms        : 헤더 슬라이드업 + 페이드인 (800ms)
600ms      : 씨앗 카운트 애니메이션 0 → N (1800ms)
1800ms     : 감성 텍스트 1 페이드인
~2800ms    : 감성 텍스트 2 페이드인
~4000ms    : 격려 메시지 페이드인 + 버튼 표시
동시 진행   : 씨앗 낙하 애니메이션 (약 8초에 걸쳐 전체 씨앗 수만큼)
씨앗 안착 후 1.5초 : 꽃 블룸 애니메이션 (2초)
```

### 씨앗 낙하 + 안착 시스템
- 화면 위에서 랜덤 위치로 생성, 천천히 떨어짐
- **텍스트 바운딩 박스 위에 30% 확률로 착지** (나머지는 통과)
- **바닥에는 100% 착지**
- 쌓이는 높이 추적: 같은 표면에 착지할수록 조금씩 쌓여 올라감
- 랜덤 x 오프셋으로 자연스러운 불규칙성

### 텍스트 (감성 메시지)
```
"호흡하며 민들레 씨앗이 [N]개 날아갔어요!"

"당신의 호흡으로 민들레의 희망이 되었어요"

"작은 숨 하나가 어딘가에서 꽃을 피울 거예요.
 오늘 당신이 내쉰 바람은 씨앗이 되어
 세상 어딘가에 조용히 뿌리를 내릴 거예요."

"잘했어요. 오늘 하루도 수고했어요. 🌱"
```

## 6. 감성 설계 아크

### 감정 곡선

```
[멈춤/안정] → [몰입/연결] → [성취/감동] → [긍정/안심]
```

| 구간 | 감정 | 구현 방식 |
|------|------|----------|
| 인트로 | 차분함, 호기심 | 어두운 블루 배경, 부드러운 타이포, 시적 내러티브 |
| 들숨 | 집중, 준비 | 카운트다운, 단순한 화면 |
| 날숨 | 몰입, 연결감 | 호흡에 실시간 반응하는 파티클/글로우/흔들림 |
| 라운드 브레이크 | 성취감 | "잘했어요!" + 씨앗 카운트 |
| 결과 | 감동, 긍정 | 시적 메시지, 꽃 블룸, 씨앗 낙하 |

### 핵심 감성 전략
1. **개인 행동의 세계적 영향**: "내 숨 하나가 꽃을 피운다"
2. **즉각적 시각 보상**: 내쉬는 순간 씨앗이 날아감
3. **정량화된 성취**: 숫자로 보여주는 "N개의 씨앗"
4. **시적 마무리**: 결과를 통해 의미를 부여

## 9. 새 호흡 게임 기획 시 재사용 가능한 패턴

### 반드시 유지할 것
- **적응형 캘리브레이션**: 들숨 구간에서 주변 소음 자동 측정
- **지속감지 필터**: 240ms 이상 지속된 소음만 호흡으로 인정
- **breathLevel (0~1)**: 하나의 정규화된 값으로 모든 시각 피드백 제어
- **감성 아크**: 차분한 시작 → 몰입 → 성취 → 긍정 마무리

### 변형 가능한 것
- **시각적 메타포**: 민들레 → 별자리, 꽃잎, 파도, 바람개비 등
- **호흡 매핑 대상**: 씨앗 → 별, 선(constellation line), 빛 입자 등
- **라운드 수 / 시간**: 3라운드 × 5초는 약 1분 세션, 조절 가능
- **결과 연출**: 낙하+안착 대신 완성된 별자리 조명, 줌아웃 등

### 별자리 게임 적용 예시 (참고)
```
들숨(3초) → 별 하나가 빛나기 시작
날숨(0~5초) → 호흡 세기에 따라 별에서 선이 뻗어나감
  - breathLevel → 선의 밝기/두께/속도
  - 씨앗 대신 "별빛 입자"가 선을 따라 이동
라운드 3회 → 별자리 하나 완성
결과: 완성된 별자리 + "오늘 당신이 그린 별자리" 메시지
```

## 10. 기술 스택

- **Flutter** + **Flame** (게임 엔진)
- **noise_meter**: 마이크 dB 측정
- **permission_handler**: 마이크 권한
- **flutter_svg**: SVG 렌더링
- **CustomPainter + RepaintBoundary**: 고성능 파티클 렌더링
- **Object Pool**: GC 없는 파티클 관리
