좋아.
아래는 네가 원한 내용을 반영해서, 기존 민들레 문서 형식에 맞춘 **랜덤 분기형 별자리 호흡 게임 전체 기획 분석서**야.

---

# 별자리 호흡 게임 — 기획 분석서

> 이 문서는 기존 민들레 호흡 게임의 구조를 바탕으로,
> **랜덤 분기형 별자리 생성**이라는 새로운 메타포로 재설계한 호흡 게임 기획안입니다.
> 핵심 목표는 유저의 호흡을 통해 밤하늘에 새로운 길을 만들고,
> 세 번의 호흡 끝에 유저만의 별자리를 완성하게 하는 것입니다.

---

## 1. 전체 유저 플로우

```text
인트로 (5페이지 스와이프) → 게임 (3라운드) → 결과 화면
```

### 인트로 화면 (세로 스와이프, 5페이지)

| 페이지       | 내용                                               | 역할         |
| --------- | ------------------------------------------------ | ---------- |
| 1. 히어로    | "CONSTELLATION BREATH" / "호흡으로 밤하늘에 별의 길을 그려보세요" | 세계관 제시     |
| 2. 스토리    | "숨을 고르면, 흩어진 별빛도 길을 찾아요"                         | 감성 동기부여    |
| 3. 방법 안내  | 마이크 가까이 → 들숨 3초 → 천천히 날숨 → 별 생성 / 3번 반복          | 인터랙션 학습    |
| 4. 마이크 권한 | "마이크는 게임 중에만 사용되며 저장되지 않아요"                      | 권한 요청 + 신뢰 |
| 5. 시작     | "준비됐다면, 오늘의 별을 이어볼까요?"                           | 진입         |

### 핵심 설계

기존 민들레와 동일하게
**스토리 → 동기 → 방법 → 신뢰 → 시작**
순으로 감정을 쌓는다.

다만 민들레가 “바깥으로 퍼짐”의 감성이라면,
별자리는 **“숨결이 밤하늘에 흔적을 남긴다”**는 감성으로 설계한다.

---

## 2. 게임의 핵심 컨셉

### 한 줄 컨셉

**들이숨으로 별빛을 모으고, 날숨으로 밤하늘에 새로운 길을 만든다.
세 번의 호흡 끝에 유저만의 랜덤 별자리가 완성된다.**

### 핵심 플레이 감각

이 게임은 미리 정해진 별들을 정확히 잇는 퍼즐이 아니다.
유저의 날숨이 **화면 하단에서 시작하여 밤하늘로 퍼져나가며**,
그 끝에서 새로운 별이 생성되고,
그중 하나가 다음 시작점으로 선택되면서
유저만의 경로가 만들어지는 **탐색형 호흡 게임**이다.

### 이 게임이 주는 감정

* 내 숨이 직접 하늘에 흔적을 만든다
* 다음에 어디로 이어질지 살짝 기대된다
* 화면이 전환되는 것이 아니라 내가 별을 따라 이동하는 느낌이 든다
* 마지막엔 “오늘의 호흡이 만든 별자리”를 내려다보게 된다

---

## 3. 전체 게임 플레이 구조

### 4단계 순환 × 3라운드

```text
[준비 3초] → [들숨 3초] → [날숨 0~5초] → [라운드 브레이크 2.2초] → 반복
```

### 3라운드의 의미

* 1라운드: 시작 별에서 첫 번째 별/분기 생성
* 2라운드: 선택된 별로 이동 후 다시 분기
* 3라운드: 마지막 분기 생성
* 종료 후: 지금까지 생성된 모든 별과 선을 줌아웃으로 보여주며 전체 별자리 완성

### 구조적 특징

민들레는 “정량화된 씨앗 수”가 핵심 보상이었고,
이번 별자리 게임은 **유기적으로 생성된 경로 자체**가 보상이다.

---

## 4. 게임 플레이 상세 구조

---

### Phase 1: 준비 (PREPARE) — 3초

#### 화면

* 어두운 밤하늘 배경
* 현재 시작 별이 화면 하단 근처에 위치
* 주변엔 아직 생성된 별들이 흐리게 남아 있거나, 첫 라운드라면 비어 있음
* 마이크 아이콘 + 부드러운 원형 가이드 표시

#### 텍스트 예시

* "입을 마이크에 가까이 대세요"
* "별빛을 이을 준비를 해요"

#### 목적

* 유저가 실제로 휴대폰 하단 마이크를 인식하게 함
* 시각적으로도 시작점이 늘 아래에서 시작한다는 규칙을 학습시킴

---

### Phase 2: 들숨 (INHALE) — 3초

#### 화면

* 현재 시작 별이 점점 밝아짐
* 별 주변의 halo가 커짐
* 별빛 입자가 안쪽으로 모여드는 느낌
* 화면의 호흡 가이드 원이 천천히 팽창

#### 텍스트 예시

* "숨을 크게 들이쉬세요"
* "별빛을 모으고 있어요…"

#### 기능적 역할

* 기존 민들레 게임처럼 **주변 소음 캘리브레이션** 수행
* 동시에 시각적으로는 “별에 에너지를 충전하는 단계”로 보이게 설계

#### 감성적 역할

민들레에서 들숨은 준비 단계였다면,
별자리에서는 **“빛이 차오른다”**는 의미를 더 강하게 부여한다.

---

### Phase 3: 날숨 (EXHALE) — 최대 5초

#### 핵심 인터랙션

* 시작 별에서 **1개 또는 2개의 선 후보**가 아래에서 위로 퍼져나감
* 이 선은 완전한 직선보다는 **직선 기반 + 아주 약한 흔들림**이 있는 형태
* 선 끝에 도달하면 어두운 점이 생기고, 잠깐 뒤 반짝이며 새로운 별 생성
* 만약 두 개의 별이 생성되었다면, 그중 하나가 랜덤 선택되어 다음 시작점이 됨

#### 텍스트 예시

* "천천히 내쉬세요"
* "별의 길이 이어지고 있어요"
* "빛이 밤하늘로 퍼져나가요"

#### 호흡과 연결되는 요소

* 날숨이 길수록 선이 더 길게 뻗음
* 선이 더 밝고 두꺼워짐
* 선을 따라 이동하는 별빛 입자 수가 많아짐
* 새로 생성되는 별의 반짝임이 더 풍부해짐

#### 중요한 UX 원칙

날숨이 짧다고 해서 아무것도 생기지 않으면 안 된다.
**별은 항상 생성되되, 호흡이 좋을수록 더 아름답게 생성**되어야 한다.

---

### Phase 4: 라운드 브레이크 — 2.2초

#### 화면

* 새로 생성된 별들이 잠깐 반짝이고 안정됨
* 선택된 별은 조금 더 강하게 강조됨
* 선택되지 않은 별도 화면상에 남아 별자리의 일부가 됨

#### 텍스트 예시

* "잘했어요!"
* "새로운 별이 태어났어요"
* "빛의 길이 더 넓어졌어요"

#### 역할

* 라운드 완료 보상
* 다음 시작점이 선택되었다는 시각적 인지
* 카메라 이동 직전의 짧은 정리 구간

---

## 5. 랜덤 분기 구조 (핵심 시스템)

이 게임의 가장 큰 차별점은
**미리 정해진 노드를 잇는 것이 아니라, 호흡마다 새로운 별이 생성되고 분기된다는 점**이다.

---

### 5-1. 분기 개수 규칙

각 날숨마다 선은 다음 중 하나로 생성된다.

* 70% 확률: 1개 분기
* 30% 확률: 2개 분기

#### 이유

* 항상 2개면 복잡해지고 시선이 분산됨
* 항상 1개면 랜덤성이 약함
* 가끔 갈림길이 등장해야 “탐색하는 느낌”이 살아남

---

### 5-2. 선의 시작점 규칙

모든 선은 **현재 시작 별**에서 시작한다.
하지만 카메라 연출 때문에 유저는 늘 그 별이 화면 하단에서 시작되는 것처럼 느낀다.

즉,

* 월드 좌표상 별은 어디에 있어도 됨
* 화면상 구도는 늘 “휴대폰 아래쪽에서 하늘로 퍼지는 느낌”을 유지해야 함

---

### 5-3. 선의 각도 규칙

선은 시작점 기준으로 **위쪽 반원 범위 안에서 랜덤하게 퍼져나간다.**

예시 범위:

* 대략 `-100° ~ +100°`
* 단, 화면 아래로 내려가는 각도는 금지
* 거의 정면 위일 수도 있고
* 좌상향 / 우상향일 수도 있고
* 때로는 거의 90도 측면에 가깝게 이어질 수도 있음

#### 의도

* 사용자는 날숨이 위로 뻗는 감각을 유지함
* 동시에 매번 다른 별자리 구조가 나옴

---

### 5-4. 선 길이 규칙

선 길이는 **호흡 길이 + 랜덤 보정**으로 결정한다.

#### 예시 구간

* 짧은 날숨: 짧거나 중간 거리
* 중간 날숨: 중간 거리 중심
* 긴 날숨: 긴 거리 중심

#### 목적

* 유저 호흡이 결과에 영향을 준다는 감각 유지
* 결과가 매번 똑같이 느껴지지 않도록 랜덤 편차 부여

---

### 5-5. 별 생성 규칙

선 끝에는 곧바로 밝은 별이 생기지 않고, 다음 단계로 생성된다.

```text
어두운 점 등장 → 미세한 깜빡임 → bloom → 안정된 별로 정착
```

#### 의도

* “길 끝에서 별이 태어난다”는 느낌 강화
* 단순 UI 생성이 아니라 감성적 탄생 연출 제공

---

### 5-6. 다음 시작점 선택 규칙

두 개의 별이 생성되었다면,
그중 하나가 랜덤하게 선택되어 다음 시작점이 된다.

#### 선택되지 않은 별의 처리

* 사라지지 않는다
* 현재 별자리의 일부로 남는다
* 결과 화면에서 전체 구조를 이룬다

#### 선택된 별의 처리

* 밝기가 조금 더 강해짐
* 짧게 1회 pulse
* 그 뒤 카메라가 해당 별로 이동

---

## 6. 카메라 시스템 (핵심 연출)

이 게임의 몰입감은 **카메라 이동**에서 크게 나온다.

중요한 점은
이동이 “화면 전환”처럼 보이면 안 되고,
**방금 내 숨으로 이어진 별을 따라 실제로 시선이 이동하는 느낌**이어야 한다.

---

### 카메라의 역할

* 선택된 별을 다음 시작점처럼 보이게 재배치
* 유저가 밤하늘을 이동하며 별을 이어간다는 감각 제공
* 라운드 사이 단절감을 줄이고 여정감을 강화

---

### 카메라 이동 규칙

#### 이동 타이밍

* 라운드 브레이크 구간 중 실행
* 별 생성 및 랜덤 선택이 끝난 뒤
* 다음 라운드 시작 전에 완료

#### 목표 위치

선택된 별이 다음 라운드에서
**화면 하단 20~30% 지점**에 오도록 카메라가 정렬된다.

#### 이동 방식

* 부드러운 pan
* 미세한 zoom in / zoom out
* 아주 약한 rotation 보정

#### 추천 범위

* 이동 시간: 600ms ~ 1000ms
* 카메라 회전: `-6° ~ +6°`
* 줌: `0.95 ~ 1.08`
* easing: 부드러운 easeInOut 계열

#### 사용자 체감

* 별을 따라 시선이 끌려감
* 새로운 출발점에 도착했다는 느낌
* 다음 호흡이 자연스럽게 이어짐

---

### 결과 화면에서의 카메라

게임 종료 후에는 반대로

* 천천히 줌아웃
* 지금까지 생성된 모든 별과 선이 한 프레임 안에 들어오게 정리
* 전체 별자리를 보여줌

즉, 게임 중 카메라는 **몰입을 위해 좁게**,
결과에선 **의미를 위해 넓게** 사용한다.

---

## 7. 호흡 감지 메커니즘

기술적으로는 민들레 호흡 게임 구조를 거의 그대로 재사용할 수 있다.

---

### 입력 소스

* `noise_meter` 패키지 사용
* 초당 약 8~10회 dB 리딩

---

### 캘리브레이션 (매 라운드 들숨 3초 동안)

```text
1. 들숨 3초간 dB 리딩 수집
2. 읽기 3개 이상: 중앙값(median)을 주변 소음 기준으로 설정
3. 읽기 1~2개: 평균값 사용
4. 읽기 0개: 폴백 52dB 사용
5. 감지 임계값 = 주변 소음 + 10dB
```

#### 의미

* 장소마다 다른 주변 소음에 자동 적응
* 하드코딩보다 훨씬 안정적
* 기존 민들레 게임의 강점을 그대로 유지

---

### 호흡 감지 알고리즘

```text
breathLevel = ((현재dB - 임계값) / 18.0).clamp(0.0, 1.0)

매 120ms마다:
  normalized = (현재dB - 임계값) / 18.0

  if normalized <= 0:
      연속감지 = 0
      호흡활성 = false
  else:
      연속감지++
      if 연속감지 >= 2:
          호흡활성 = true
```

---

### 핵심 상수

| 상수                       |         값 | 의미              |
| ------------------------ | --------: | --------------- |
| `ambientOffset`          |   10.0 dB | 주변 소음 대비 감지 오프셋 |
| `breathRange`            |   18.0 dB | 강도 정규화 범위       |
| `fallbackThreshold`      |   52.0 dB | 캘리브레이션 실패 시 기본값 |
| `sustainedTicksRequired` | 2 (240ms) | 오탐 방지 최소 지속 시간  |

---

### 오탐 방지 설계

* 240ms 지속 필터
* 라운드별 재캘리브레이션
* 10dB 오프셋
* 2초 이상 미감지 시 힌트 표시

---

## 8. 시각 피드백 시스템

별자리 게임에서도 핵심은 `breathLevel (0.0 ~ 1.0)` 하나로
여러 시각 요소를 동시에 제어하는 것이다.

---

### breathLevel이 제어하는 요소

| 요소         | 최소    | 최대     | 의미              |
| ---------- | ----- | ------ | --------------- |
| 선 뻗는 속도    | 느림    | 빠름     | 날숨이 길수록 빠르게 연결  |
| 선 밝기       | 희미함   | 선명함    | 호흡 품질에 따른 빛의 강도 |
| 선 두께       | 가늘음   | 약간 두꺼움 | 안정감 있는 연결 느낌    |
| 별빛 입자 수    | 적음    | 많음     | 선 위를 흐르는 빛의 양   |
| 시작 별 glow  | 약함    | 강함     | 들숨 충전감          |
| 새 별 bloom  | 작음    | 큼      | 생성 보상감          |
| 배경 성운 glow | 거의 없음 | 은은히 확장 | 공간 전체 반응        |

---

### 들숨이 제어하는 요소

들이숨은 “모으는 단계”이므로 아래 요소를 강화한다.

* 시작 별 pulse 크기
* 시작 별 주변 halo 반경
* 별빛 입자의 inward motion
* 다음 날숨 시작 시 선의 초기 광도

---

### 날숨이 제어하는 요소

날숨은 “펼치는 단계”이므로 아래 요소를 강화한다.

* 선의 길이
* 선의 밝기
* 선의 두께
* 선 위를 흐르는 입자량
* 새 별 생성 이펙트
* 가끔 2개 분기 발생 가능성 증가

---

### 추천 렌더링 구조

```text
[배경 성운 글로우] → [별빛 먼지 파티클] → [연결선 트레일] → [별 노드 + UI]
```

---

#### 1) 배경 성운 글로우

* 현재 시작 별 뒤에 은은한 radial glow
* 호흡이 감지되면 강해짐
* 색상은 푸른빛/보랏빛 계열

#### 2) 별빛 먼지 파티클

* 선을 따라 이동하는 반짝 입자
* 작고 부드럽게 흐름
* 씨앗 대신 “빛이 이동한다”는 메타포 역할

#### 3) 연결선 트레일

* 선이 실시간으로 자라남
* 머리 부분이 이동하고 뒤로 잔상이 남는 느낌
* 호흡이 길수록 더 안정적이고 밝음

#### 4) 별 노드

* 시작 별 / 생성된 별 / 선택된 별 구분
* 선택된 별은 잠깐 강조
* 새 별 생성 시 bloom 연출

## 9. 별자리 구조 저장 방식

이 게임의 결과는 단순한 한 줄 경로가 아니라,
**작은 분기가 남아 있는 유기적 구조**가 된다.

### 예시

```text
1회차: A → B, C
선택: B
2회차: B → D
3회차: D → E, F
선택: E
```

최종적으로 화면에는

* A-B
* A-C
* B-D
* D-E
* D-F

가 남을 수 있다.

즉 결과 화면에서 보여주는 별자리는
**하나의 직선이 아니라, 사용자의 호흡이 남긴 분기 흔적을 포함한 구조**다.

이게 이 게임만의 개성이 된다.

## 10. 결과 화면

### 연출 타임라인

```text
0ms      : 마지막 생성 별 안정화
400ms    : 카메라 멈춤
600ms    : 전체 화면 줌아웃 시작
1600ms   : 전체 별자리 전부 화면 안에 들어옴
1800ms   : 별과 선이 순서대로 점등
2500ms   : 별자리 이름 등장
3200ms   : 감성 메시지 등장
4000ms   : 버튼 표시
```

---

### 결과 화면에서 보여줄 것

* 지금까지 생성된 모든 별
* 연결된 모든 선
* 분기된 흔적까지 포함된 전체 구조
* 오늘의 별자리 이름
* 감성 메시지
* 저장 / 다시 하기 버튼

---

### 결과 카피 예시

* "오늘 당신의 호흡이 만든 별자리예요"
* "흩어진 빛이 하나의 길이 되었어요"
* "작은 숨 하나가 밤하늘에 흔적을 남겼어요"
* "당신의 호흡이 오늘의 별을 이었어요"

---

### 별자리 이름 생성 아이디어

랜덤 경로 기반으로 감성 이름을 붙일 수 있다.

예시:

* 푸른 갈림길
* 고요한 숨결
* 밤의 나침반
* 은은한 파문
* 작은 항로
* 새벽의 분기점

---

## 11. 감성 설계 아크

### 감정 곡선

```text
[정적] → [집중] → [확장] → [탐색] → [경이] → [안심]
```

| 구간    | 감정      | 구현 방식           |
| ----- | ------- | --------------- |
| 인트로   | 고요함, 기대 | 밤하늘, 시적 카피      |
| 들숨    | 집중, 충전  | 별빛이 모이는 시각 효과   |
| 날숨    | 확장, 탐색  | 선이 퍼지고 별이 태어남   |
| 분기 선택 | 기대감, 몰입 | 랜덤 선택 + 카메라 포커싱 |
| 결과    | 경이, 안심  | 줌아웃 + 전체 별자리 공개 |

---

### 핵심 감성 전략

1. **호흡이 단순한 숨이 아니라 길이 된다**
2. **내 행동이 예측 불가능하지만 아름다운 결과를 만든다**
3. **화면 전환이 아니라 여정처럼 이어진다**
4. **마지막에 전체 구조를 보며 의미를 느낀다**

## 16. MVP 기준 구현 권장안

처음 버전은 아래처럼 단순화하는 것이 좋다.

### MVP 구조

* 인트로 5페이지 유지
* 3라운드 유지
* 1회당 1~2개 분기
* 총 별 수 4~6개 수준
* 선은 직선 기반
* 카메라 회전은 아주 미세하게만
* 결과 화면에서는 전체 줌아웃 + 이름 표시만 우선 구현

### 이유

* 기존 민들레 구조 재사용성이 높음
* 랜덤성이 있으면서도 복잡하지 않음
* 유저에게 충분히 새로움
* 구현 난이도와 연출 퀄리티 균형이 좋음
